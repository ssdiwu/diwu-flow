# 上位心智层

> **规则约束级别说明**：本文件定义上位心智层的核心规则。除非特别标注 `[建议]`，否则都是必须遵守的约束。

## 三唯一框架

进入任务前必须先确认三件事，缺一不可：

| 要素 | 定义 | 常见错误 |
|------|------|---------|
| **唯一主线目录** | 本轮任务唯一允许承接主要实现、验收和结论归属的目录 | 沿旧副本、旧入口、旧文档推进 |
| **唯一运行入口** | 验证主链是否真的经过时，实际使用的启动、调用或工作入口 | 用不存在的脚本或过时的命令当入口 |
| **canonical（规范真相源）** | 当前必须以其为准的说明、接口、任务定义或规则文件 | 与旧副本并列引用 |

未确认三唯一前，不要默认当前目录、旧脚本、旧副本就是主线。

## 现象→判断→动作思维骨架

所有规则都是这条链的具体实例。违反此链的规则是空壳。

| 环节 | 含义 | 常见缺失 |
|------|------|---------|
| **现象** | 看到了什么（事实、数据、异常） | 抽象描述代替具体观察 |
| **判断** | 据此得出什么结论（依据什么） | 跳步到动作，或缺少依据 |
| **动作** | 接下来要做什么（具体行动） | 停在理解层，不推动执行 |

**输出骨架模板**：
```
现象：- 我现在看到的事实
判断：- 基于事实我认为问题更像什么；已排除哪些误判
动作：- 下一步具体做什么；怎样验证
```

这条框架优先于任何具体规则。规则与框架冲突时，先检查三段论是否完整。

> 完整五问开工检查与误判表预加载见执行引擎 SKILL.md §Preflight。

## 不确定性门控

任务入口处根据不确定性选择路径：

### 可直接做（同时满足三项）

| 条件 | 说明 |
|------|------|
| 改动范围小 | 预估 < 200 行或单点修改 |
| 结果可预期 | 团队做过类似事，把握 > 90% |
| 预期差异可一句话说清 | 无需长文描述预期变化 |

### 必须先写最小规格（满足任一）

| 触发条件 | 说明 |
|---------|------|
| 结果不能一句话说清 | 预期复杂或多分支 |
| 依赖外部系统 / 运行时链路 / 跨层集成 | 引入不可控变量 |
| 输出需稳定格式 / 字段 / 验收标准 | 有契约约束 |
| 需交接给他人或下一轮继续 | 跨 session 连续性要求 |

### 先探索验证（满足任一）

| 触发条件 | 说明 |
|---------|------|
| 外部依赖多且不可控 | 第三方 API、LLM 行为等 |
| 真实落点不清楚 | 需先缩小问题范围 |
| 组合行为不稳定 | 多模块集成未测试过 |
| 回滚成本高 | 方向错了代价大 |

> 判断锚点（正例/反例/边界例）见 `rules/judgments.md`。
