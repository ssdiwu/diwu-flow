# 工作流补充规则

> 本文件只保留 Skills 未覆盖的独特内容。主体回答：从哪一层进入哪一层、什么时候该转层、什么时候该回退。
> 任务规划/实施/提交/子代理策略/验证要求分别见任务规划工具 / 执行引擎。
> 子代理交接协议见 `rules/handoff.md`，测试分层策略见 `rules/testing.md`。

## 一、现象→判断→动作思维框架（快速参考）

完整定义见 `mindset.md` §现象→判断→动作。此处为快速查阅版。

| 环节 | 含义 | 常见缺失 |
|------|------|---------|
| **现象** | 看到了什么（事实、数据、异常） | 抽象描述代替具体观察 |
| **判断** | 据此得出什么结论（依据什么） | 跳步到动作，或缺少依据 |
| **动作** | 接下来要做什么（具体行动） | 停在理解层，不推动执行 |

**输出骨架模板**：
```
现象：- 我现在看到的事实
判断：- 基于事实我认为问题更像什么；已排除哪些误判
动作：- 下一步具体做什么；怎样验证
```

这条框架优先于任何具体规则。规则与框架冲突时，先检查三段论是否完整。

## 二、阶段流转速查

### 层级路线图

```
Plan → (≥3 步?) → 任务定义文件(InDraft) → InSpec → 执行引擎(InProgress) → InReview → Done
                     ↑                                           |
                     └─────── <3 步直做 ←────────────────────────────┘
```

### 入口门控

| 入口 | 条件（满足任一即走此路径） |
|------|--------------------------|
| **直做** | 改动范围小 + 结果可预期 + 预期差异一句话说清 |
| **先写任务定义** | 结果不能一句话说清 / 依赖外部系统 / >3 步 / 需跨 session |
| **先探索** | 外部依赖多且不可控 / 真实落点不清楚 / 回滚成本高 |

### 跨阶段回退

| 当前 | 事件 | 回退到 | 规则 |
|-------|------|--------|------|
| InProgress | 发现 acceptance 有误 | InSpec | 记录原因，重新确认需求 |
| InReview | FAIL 条目存在 | InProgress | 按 FAIL 原因修复后重新验证 |
| InProgress | 环境阻塞 | InSpec + BLOCKED | 见 `rules/exceptions.md` |

> 完整不确定性决策见 `mindset.md` §不确定性门控。完整 Plan→Dtask 门控见 `rules/handoff.md` §七。
