### 公开版本发布流程备忘（重要！）

#### 前置配置（只需一次）
```bash
# 在 diwu-flow 项目根目录下：
git remote add public git@github.com:ssdiwu/diwu-flow-public.git
```

#### 每次大版本发布步骤
```bash
# 1. 确保 main 分支工作区干净，代码已 commit
# 2. 执行发布脚本：
./drelease.sh v1.0.0 --push-public

# 脚本会自动完成：
#   - 从 main 创建 release/v1.0.0 分支
#   - git rm 排除敏感文件（dtask.json、.diwu/、recording/、continue-here.md、decisions.md）
#   - 打 tag v1.0.0
#   - 推送私有仓库的 release 分支和 tag
#   - 推送到公开仓库的 main 分支和 tag
#   - 自动切回 main
```

#### 排除的文件清单（.gitattributes 定义）
| 文件 | 原因 |
|------|------|
| `.diwu/` | 运行时状态目录 |
| `dtask.json` | 任务数据（含内部任务信息） |
| `recording/` | session 记录（含内部讨论） |
| `continue-here.md` | session 断点恢复临时文件 |
| `decisions.md` | 设计决策记录（可能含内部讨论） |
| `.env` / `*.log` | 环境变量与日志 |

#### 私有 vs 公开仓库关系
```
ssdiwu/diwu-flow (private)     ← 日常开发，频繁小更新
  ├── main                     ← 工作分支
  └── release/v*               ← 公开镜像分支（已清理）

ssdiwu/diwu-flow-public (public) ← 仅接收干净的 release 分支
  └── main                     ← 来自 release/v*:main
```
