# 状态文件规格

## dtask.json — 任务定义与状态真相源

### 字段结构

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `tasks` | array | 是 | 任务列表，每个任务为对象 |
| `tasks[].id` | int | 是 | 从 1 递增，永不复用 |
| `tasks[].title` | string | 是 | 一句话描述（动词开头） |
| `tasks[].description` | string | 是 | 背景 + 关键约束 |
| `tasks[].acceptance` | array | 是 | GWT 格式验收场景 |
| `tasks[].steps` | array | 是 | 实施步骤（绝对路径） |
| `tasks[].files_modified` | array | 否 | 并行冲突检测 |
| `tasks[].category` | string | 否 | functional/ui/bugfix/refactor/infra |
| `tasks[].blocked_by` | array[int] | 否 | 前置任务 ID 列表 |
| `tasks[].status` | string | 是 | 运行时状态（见状态机） |

### 状态机

```
InDraft ──(人工确认)──> InSpec ──(开始实施)──> InProgress
  │                       │                        │
  │ (取消)                 │ (发现问题)              │ (遇阻塞)
  ↓                       ↓                        ↓
Cancelled                InSpec                   InSpec
                                                   │
                                              (完成准备验证)
                                                   ↓
                                                InReview
                                               /   |    \
                                         (小幅度) (大幅度) (失败返工)
                                            |       |       |
                                            v       v       v
                                          Done    Done   InProgress
                                                  (需确认)
```

### 状态迁移命令（dtask_transition.py）

| 命令 | 源状态 | 目标状态 | 写入字段 |
|------|--------|---------|---------|
| `mark-inspec` | InDraft | InSpec | `task.status` + sync runtime |
| `claim` | InSpec | InProgress | `task.status` + `task_sessions[owner]` |
| `release --to inreview` | InProgress | InReview | `task.status` + clear owner + pending_recording |
| `release --to done` | InProgress | Done | `task.status` + clear owner + pending_recording |
| `release --to inspec` | InProgress | InSpec | `task.status` + clear owner + pending_recording |
| `release --to cancelled` | InProgress | Cancelled | `task.status` + clear owner + pending_recording |
| `adopt` | InProgress | InProgress | `task_sessions[owner]`（更换 session） |
| `show-pending` | — | — | 只读（**persist=True** 副作用：sync_runtime_state 写盘） |
| `clear-pending` | — | — | 清除 pending_recording 标记 |
| `sync-runtime` | — | — | 只读（**persist=True** 副作用：cleanup + legacy 迁移写盘） |

## dtask-state.json — Runtime 元数据真相源

> **不重复保存 task status**。status 以 `dtask.json` 为唯一真相源。

### 顶层结构

```json
{
  "version": 1,
  "task_sessions": { "<task_id>": { "session_id": "...", "started_at": "..." } },
  "dloop": null,
  "pending_recording": null
}
```

### Writer Matrix（全部写者）

| 写者 | 写入字段 | 触发条件 | persist |
|------|---------|---------|---------|
| `dtask_transition.py cmd_claim` | `task_sessions[task_id]`（set_task_owner） | claim 命令 | save_runtime_state |
| `dtask_transition.py cmd_release` | `task_sessions[task_id]`（clear_task_owner）+ `pending_recording`（set） | release 命令 | save_runtime_state |
| `dtask_transition.py cmd_adopt` | `task_sessions[task_id]`（set_task_owner，替换 session） | adopt 命令 | save_runtime_state |
| `dtask_transition.py cmd_show-pending` | 无逻辑写入（**persist=True** 副作用：sync_runtime_state cleanup 写盘） | show-pending 查询 | sync_runtime_state(persist=True) |
| `dtask_transition.py cmd_sync-runtime` | cleanup stale owners + legacy dloop 迁移 | sync-runtime 命令 | sync_runtime_state(persist=True) |
| `task_completed.py` | `task_sessions[task_id]`（clear_task_owner） | TaskCompleted 事件 | save_runtime_state |
| `task_completed.py` | `dloop.completed_task_ids`（追加） | TaskCompleted 事件 + dloop active | sync_runtime_state(persist=True) + save_runtime_state |
| `stop_decision.py decide_loop_mode` | `dloop.active=false` + `stopped_at` + `stop_reason`（clear_loop_state） | dloop 终止条件命中 | save_runtime_state |
| `stop_decision.py decide_loop_mode` | `dloop.current_iteration++` | dloop 未终止时迭代 | save_runtime_state |
| `stop_decision.py main` | `dloop.session_id` 替换（dummy→真实 SID） | 首次 Stop 绑定 | save_runtime_state |
| `dloop.py stop` (cmd_stop) | `dloop` 整体清除（clear_loop_state） | `/dstop` 停止循环 | sync_runtime_state(persist=True) + save_runtime_state |
| `dloop_state.py cleanup_state` | `dloop` 整体清除（clear_loop_state） | 外部清理调用 | sync_runtime_state(persist=True) + save_runtime_state |

### Self-Heal 语义（sync_runtime_state）

`sync_runtime_state()` 在每次被调用时执行以下自动修复：

1. **Legacy 迁移**：若 `.diwu/dloop-state.json` 存在且当前 `dloop` 为 null，自动迁移并删除旧文件
2. **Stale owner 清理**：`task_sessions` 中指向非 InProgress 任务的 owner 条目自动删除
3. **Pending recording 自愈**：若 `pending_recording` 指向的任务不存在或状态不匹配目标，自动清除标记
4. **Persist 门控**：仅当 changed 或 ensure_exists 或 migrated_legacy_loop 为真时才写盘（除非显式传 persist=True）

## dsettings.json — 可调参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `task_archive_threshold` | int | 20 | Done/Cancelled 任务数超此阈值触发归档 |
| `recording_archive_threshold` | int | 30 | session 文件数超此阈值触发归档 |
| `recording_reminder.enabled` | bool | true | TaskCompleted 后是否输出 recording 提醒 |
| `error_tracking.enabled` | bool | true | 3-Strike 工具失败追踪是否启用 |
| `review_limit` | int | 5 | dloop 超前实施上限 |
