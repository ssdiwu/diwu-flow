# .doc/ 导航索引

> 本目录是 diwu-flow 的**设计文档真相源**：架构解释、设计原理、数据规格。
> 不复读 `README.md`（用户视角）或 `rules/`（运行规则正文）的内容。

## 文档索引

| 文件 | 回答什么 | 何时看 |
|------|---------|--------|
| [架构规范.md](./架构规范.md) | 系统由哪些层组成、模块如何协作、drun dual-entry、Persistence Policy | 想理解整体骨架或查模块边界 |
| [状态文件规格.md](./状态文件规格.md) | dtask.toml / dtask-state.toml 字段语义与写者矩阵 | 要读写状态文件或调试状态不一致 |
| [工程规范.md](./工程规范.md) | 变更传播规则、禁止事项、引用索引 | 要改代码前确认约束和影响范围 |

## 推荐阅读顺序

1. **架构规范** — 先建立六层模型全局图
2. **状态文件规格** — 理解数据真相源的读写契约
3. **工程规范** — 了解修改时的约束和传播路径

## 维护规则

- 本目录不维护静态 inventory（如 Command/Skill 计数），计数以 `.claude-plugin/plugin.json` 为准
- 新增文档时更新本索引表
- 与 `rules/` 的分工：本目录承载**设计层**知识，`rules/` 承载**运行规则**正文
