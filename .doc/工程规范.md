## 禁止事项

| # | 禁止行为 | 原因 | 替代方案 |
|---|---------|------|---------|
| 1 | 在描述文件（commands/*.md）中嵌入可执行代码 | 违反 Business 约束：描述与实现分离 | 逻辑放入 scripts/ 或 skills/ |
| 2 | 手写 session 文件时间戳 | 违反时间戳铁律 | 必须运行 `date '+%Y-%m-%d %H:%M:%S'` 获取 |
| 3 | 将 dtask.json status 变更单独 commit | 违反提交原子性铁律 | 通过 `/drec` 统一完成 recording + 全量 commit |
| 4 | 在 Skill frontmatter 中使用平台专属字段 | 违反零平台耦合原则 | 使用纯流程语言描述 |
| 5 | 直接修改 `.diwu/dtask-state.json` 绕过 dtask_transition.py | 破坏 runtime 真相源一致性 | 通过 `dtask_transition.py` 的子命令操作 |
| 6 | 在 `.doc/` 中维护 Command/Skill 静态计数 | 计数以 plugin.json 为准，静态 inventory 易过期 | 引用 plugin.json 或使用 dstat 查询 |

## 变更传播矩阵

> 修改某类文件后，必须同步更新的目标位置。

| 修改源 | 传播目标 | 同步方式 | 备注 |
|--------|---------|---------|------|
| `rules/*.md` | `.claude/rules/*.md` + `assets/dinit/assets/rules/*.md` | 内容完全一致（cp/sync） | Rules 同步铁律：三副本必须一致 |
| `plugin.json` version | `marketplace.json` version + `install.sh` stub version | 三处版本号同步 | 版本号以 plugin.json 为真值来源 |
| `skills/*/SKILL.md` depends/triggers | `commands/*.md` 的引用检查 | 交叉验证 | 删除 Skill 时清理 commands 依赖 |
| `hooks/hooks.json` | `.doc/架构规范.md` Hooks 实现链表 | 手动更新 | 新增/删除 hook 事件时同步 |
| `scripts/dtask_transition.py` 子命令 | `.doc/状态文件规格.md` Writer Matrix | 手动更新 | 新增迁移命令时同步 |
| `scripts/*` (新增 writer) | `.doc/状态文件规格.md` Writer Matrix | 手动更新 | 新增 dtask-state.json 写者时追加行 |
| `.doc/` 下文件 | `rules/file-layout.md` + `.claude/rules/file-layout.md` + `assets/dinit/assets/rules/file-layout.md` | 三处 .doc/ 结构描述同步 | 本任务已建立，后续新增文档时执行 |
