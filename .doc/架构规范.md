# 架构规范

## Skills <-> Commands 映射

> Command 是薄壳（~15-30 行），Skill 是自包含规范源。

| Command | 对应 Skill | 职责 |
|---------|-----------|------|
| `/drun` | `skills/drun/SKILL.md` | 任务执行循环（Preflight -> S1-S4 -> closeout） |
| `/dtask` | `skills/dtask/SKILL.md` | 任务 CRUD + 状态机管理 |
| `/dinit` | （内嵌于 `scripts/dinit.py`） | 环境初始化与验证 |
| `/dprd` | `skills/dprd/SKILL.md` | 产品论证（竞品/画像/迭代层次/业务指标） |
| `/ddoc` | `skills/ddoc/SKILL.md` | 文档工具（正向/逆向两种模式） |
| `/drec` | `skills/drec/SKILL.md` | 记录归档唯一入口（recording + commit + 归档） |
| `/dref` | `skills/dref/SKILL.md` | 需求细化清单（Phase 1-4，止于对话层） |
| `/dcorr` | `skills/dcorr/SKILL.md` | 纠偏体系（退化信号检测 + 四行重写） |
| `/dstat` | `skills/dstat/SKILL.md` | 状态查询（任务列表/归档状态/runtime 状态） |
| `/dloop` | `skills/dloop/SKILL.md` | 连续循环启动器 |
| `/dstop` | （无独立 Skill） | 停止 dloop 循环（`scripts/dloop.py stop`） |

## 模块边界

```
agents/          -- 能力容器（explorer/implementer/verifier），默认路径自动发现
  |-- 不持有业务逻辑，只定义职责和负面清单
commands/        -- 用户入口薄壳（frontmatter + 参数检测 + 指向 Skill）
  |-- 不含执行步骤细节
skills/          -- 方法论规范源（自包含 SKILL.md）
  |-- 零平台耦合（frontmatter 无 context/agent/model/hooks）
scripts/         -- 共享脚本工具库 + script-backed 执行通道
  |-- common.py: load_json/save_json/plugin_root/max_task_id 等
hooks/           -- CC 框架事件驱动钩子
  |-- hooks.json: 声明式注册 6 个事件
rules/           -- 方法论规则文件（12 文件，由 UserPromptSubmit hook 注入）
assets/dinit/    -- dinit 模板资产（复制到用户项目）
```

## Hooks 实现链

> 基于 `hooks/hooks.json` 的 **6 个真实事件**。不含不存在的 UserPromptSubmit / PostToolUseFailure。

### 事件总览

| # | 事件 | 触发时机 | Hook 脚本（经 run_hook.py 包装） | 写入目标 |
|---|------|---------|-----------|---------|
| 1 | **SessionStart** | Session 启动时 | `session_start.py` | 注入 pitfalls 摘要 + 写 scoped session 文件 |
| 2 | **TaskCreated** | 任务创建后 | `task_created_validate.py` | 校验 dtask.json 合法性（只读） |
| 3 | **PreToolUse (Bash)** | Bash 工具调用前 | `drift_detect_pre.py` + `context_monitor.py` | 漂移检测 + 上下文监控 |
| 4 | **PreToolUse (ExitPlanMode)** | 退出 Plan 模式时 | `plan_exit_hint.py` | 输出提示（只读） |
| 5 | **PreToolUse (Edit\|Write)** | 文件写入前 | `task_entry_guard.py` | 实施入口守卫（只读校验） |
| 6 | **TaskCompleted** | 任务标记 Done 后 | `task_completed.py` | 清 owner + dloop 追踪 + reminder |
| 7 | **Stop** | Session 停止时 | `stop_decision.py` | 继续判定 / dloop 迭代 / 归档检查 / pending_recording 门控 |
| 8 | **PreCompact** | Context 压缩前 | `pre_compact.py` | 写 Checkpoint 到 recording |

### 关键数据流

```
SessionStart
  ├── 读取 project-pitfalls.md → 生成摘要 → additionalSystemPrompt 注入
  └── 写 /tmp/.claude_main_session_<repo_hash>（scoped session 文件，原子写入）

PreToolUse(Bash)
  ├── drift_detect_pre.py: 检测 .diwu/ 文件漂移
  └── context_monitor.py: 监控 context 使用量 → CRITICAL 时触发 Checkpoint

TaskCompleted (task_completed.py)
  ├── 阶段1: clear_task_owner(raw load) → _track_loop_completion(sync+append)
  └── 阶段2-3: recording_reminder 门控 → 输出 additionalSystemPrompt

Stop (stop_decision.py)
  ├── decide_default_mode: InProgress match → block / 可执行任务提示 / reminders
  ├── decide_loop_mode: stop_reason 判定 → clear_loop_state 或 iteration++
  ├── _check_pending_recording_gate: atomic commit 强制门控
  └── stop_archive.check: 归档阈值检查

执行顺序依赖（dloop 模式）:
  task_completed.py (TaskCompleted) → 写 completed_task_ids
  stop_decision.py   (Stop)            → 读 completed_task_ids + iteration 递增
  两个 hook 由 CC 框架在不同事件触发，顺序不确定。
  stop_decision.py 内置 fallback：扫描 dtask.json Done 任务补充。

### run_hook.py 包装层语义

所有 hook 脚本统一经 `hooks/scripts/run_hook.py` 执行，调用格式：

```
python3 run_hook.py --event <事件名> --script <脚本名> --mode <strict|tolerant>
```

- **`--mode strict`**（阻断级）：保留子脚本原始退出码，非零退出传递给 CC Hook 框架 → 主流程中止。适用于 `task_entry_guard`、`task_created_validate`、`task_completed`、`stop_decision`。
- **`--mode tolerant`**（提示级）：子脚本异常时降级为 exit 0，主流程继续。但 stderr 输出统一追加 `[事件/脚本名]` 前缀并写入 `.diwu/logs/hooks.log`。适用于 `drift_detect_pre`、`context_monitor`、`plan_exit_hint`、`pre_compact`、`session_start`。
```
