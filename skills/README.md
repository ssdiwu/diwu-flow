# Skills 导航

> diwu-flow 的 11 个方法论 Skill，按能力层分组。
> 回答：**用户当前意图 → 应用哪个 Skill**。
> 完整列表以 `.claude-plugin/plugin.json` 为真值来源。

## 用户意图 → Skill 选择

| 你想做什么 | 用哪个 Skill | 说明 |
|-----------|-------------|------|
| **有个想法，先挂住** | `didea` | 入口容器：捕获灵感，可选持久化到 `.diwu/ideas/` |
| **值不值得做？方向对不对？** | `dpth` | 产品思维：诊断/创始人/构建器三模式判断 |
| **需求模糊，要细化** | `dref` | 需求细化：问题发现 + 场景收敛 → 可执行清单 |
| **要写 PRD 或做产品论证** | `dprd` | 产品论证：灵魂三问门控 + 框架内化 → PRD 产出 |
| **要写产品文档** | `ddoc` | 文档工具：正向(需求→文档) / 逆向(代码→文档) / ADR |
| **要规划任务** | `dtask` | 任务管理：GWT 验收 + task.json + 规划分解 |
| **要执行任务** | `drun` | 单任务执行器：Preflight → 实施 → 验证 → 记录 |
| **要连续跑多个任务** | `dloop` | 连续循环：`while(未停止){ /drun }` 薄壳包装 |
| **跑偏了，要拉回来** | `dcorr` | 纠偏恢复：退化信号检测 + 四行重写模板 |
| **要看项目全貌** | `dstat` | 状态快照：任务/Session/决策/Git 只读聚合 |
| **要记录本次工作** | `drec` | Session 归档：格式模板 + 踩坑记录 + 原子 commit |

## 按能力层分组

### L0 — 入口 / 持久化层
要不要先挂住？要不要长期追踪？

| Skill | 定位 | 下游衔接 |
|-------|------|---------|
| `didea` | 想法捕获容器 | → dpth / dref / dprd / dtask |
| `dtask` | 任务定义与管理 | → drun |
| `drec` | Session 记录归档 | drun / dloop 的收尾入口 |

### L1 — 思考 / 收束层
值不值得做？怎么想清楚？怎么写出来？

| Skill | 定位 | 配合关系 |
|-------|------|---------|
| `dpth` | 产品思维协作（方向判断） | 与 dref/dprd 配合：dpth 先做判断，再细化/论证 |
| `dref` | 需求细化（问题发现+场景收敛） | 依赖 dpth 方向判断；输出→dtask 转化 |
| `dprd` | 产品论证（PRD 产出） | 依赖 dpth 方向判断；输出→ddoc 补充技术文档 |
| `ddoc` | 产品文档（正向/逆向/ADR） | 接收 dprd 产出；也可独立使用 |

### L2 — 执行 / 控制层
已经够清楚了，现在怎么开始执行？

| Skill | 定位 | 特殊说明 |
|-------|------|---------|
| `drun` | 单任务执行器 | 支持 dual-entry：dtask 来源 + direct request |
| `dloop` | 连续循环包装 | 最典型纯薄壳 skill，自身无执行逻辑 |

### L3 — 观察 / 纠偏层
现在发生了什么？偏没偏？

| Skill | 定位 | 触发条件 |
|-------|------|---------|
| `dstat` | 项目状态只读聚合 | 任何需要了解全局状态的时机 |
| `dcorr` | 纠偏恢复协议 | 检测到退化信号时启动（6 类泛化模式） |

## Skill 属性速查

- 所有 Skill frontmatter 无平台专属字段（零平台耦合）
- 类型分布：rule(7) / product(2) / tool(2)
- `dstop` 不是 Skill——它是 command-only 特例（无 SKILL.md）
- 完整方法论见各 `skills/{name}/SKILL.md`
