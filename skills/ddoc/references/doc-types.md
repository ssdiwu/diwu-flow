# 8 类文档类型完整定义

> 从 ddoc SKILL.md §二 类型路由层展开。每类含定义、适用场景、关键要素、模板引用、完整性检查参照。

## T1 产品需求（PRD）

**定义**：描述"为什么做"和"做什么"的产品级文档，连接业务需求与实现决策。读者是产品经理和开发者。

**适用场景**：新功能立项、需求文档化、跨团队对齐。

**关键要素**：
- 用户故事（角色 + 目标 + 动机）
- 约束矩阵（业务/时序/跨端/并发/感知）
- 验收场景（GWT 格式）
- 非功能需求

**模板引用**：`templates.md` §模块设计文档模板（约束 + 原子 + 边界 + 降级路径）

**完整性检查参照**：`completeness.md` §第一层 产品层 + 第二层 五约束维度

## T2 设计规范

**定义**：描述"技术上怎么做"的方案文档，将需求转化为可实现的架构设计。读者是开发者。

**适用场景**：技术方案评审、架构决策记录、跨模块协调。

**关键要素**：
- 约束表（五维约束展开）
- 原子分解（实体/状态/变换/事件）
- 状态机 + 管道图
- 边界定义（领域/平台/时间）
- 降级策略 + 同步策略

**模板引用**：`templates.md` §模块设计文档模板（全结构）

**完整性检查参照**：`completeness.md` §第二层 深度检查

## T3 页面规格

**定义**：单个页面/屏幕的完整规格，描述组件、交互和数据绑定。读者是前端开发人员。

**适用场景**：UI 开发前对齐、设计稿转规格、组件库文档化。

**关键要素**：
- 组件树（组件层级 + 父子关系）
- 交互状态（加载/空态/错误/正常/边界）
- 数据绑定（字段 ↔ 组件映射）
- 路由与导航

**模板引用**：`templates.md` §模块设计文档模板（边界 + 约束中感知维度 部分适配）

**完整性检查参照**：`completeness.md` §第一层 产品层 + 数据层（查询模式与数据绑定）

## T4 数据模型

**定义**：实体、字段、关系、约束的完整定义。读者是后端开发人员和 DBA。

**适用场景**：数据库设计、ORM 还原、数据迁移规划。

**关键要素**：
- 实体列表 + 字段定义（名称、类型、约束）
- 实体关系图（erDiagram）
- 主键/外键/唯一约束
- 查询模式 + 索引建议

**模板引用**：`templates.md` §模块设计文档模板（原子 + 同步策略）

**完整性检查参照**：`completeness.md` §第一层 数据层

## T5 API 契约

**定义**：端点列表 + 请求/响应格式 + 错误码的完整规范。读者是调用方（前端/第三方）。

**适用场景**：接口文档补全、第三方对接、前后端联调。

**关键要素**：
- 端点列表（路径 + HTTP 方法）
- 入参格式（请求体 + 查询参数 + 路径参数）
- 出参格式（响应体结构）
- 认证方式（JWT/OAuth/API Key）
- 错误码与含义

**模板引用**：`templates.md` §模块设计文档模板（边界 部分适配）

**完整性检查参照**：`completeness.md` §第一层 接口层

## T6 变更记录

**定义**：按版本/时间组织变更条目，描述"改了什么 + 影响谁 + 怎么迁移"。读者是开发者和运维人员。

**适用场景**：版本发布、changelog 生成、遗留系统迁移文档。

**关键要素**：
- 变更条目（新增/修改/废弃/删除）
- 影响范围（API/DB/配置）
- 迁移指南（前后兼容性 + 回退方案）
- 版本号对应关系

**模板引用**：`templates.md` §模块设计文档模板（版本兼容 部分）

**完整性检查参照**：`completeness.md` §第一层 业务逻辑层（变化路径）

## T7 运维手册

**定义**：部署拓扑、配置项、操作 SOP、告警及故障排查指南。读者是运维人员（SRE）。

**适用场景**：部署文档、故障排查手册、onboarding 运维。

**关键要素**：
- 部署拓扑图（服务依赖 + 网络架构）
- 配置项清单 + 环境变量
- 操作 SOP（启动/停止/扩容/回滚）
- 告警规则 + 故障排查流程

**模板引用**：`templates.md` §模块设计文档模板（降级路径 部分适配）

**完整性检查参照**：`completeness.md` §第一层 产品层（非功能需求）

## T8 知识问答

**定义**：以 Q&A 对为主的知识库，解答高频问题和概念。读者是新人或跨团队开发者。

**适用场景**：FAQ、onboarding 文档、概念解释、常见坑点记录。

**关键要素**：
- Q&A 对（问题 + 答案 + 代码示例）
- 概念索引（术语 → 定义映射）
- 交叉引用（链接到其他文档类型）
- 来源标注（代码/文档/经验）

**模板引用**：参考 `templates.md` §反模式速查（常见坑点）和 `templates.md` §模块设计文档模板 最小适配

**完整性检查参照**：`completeness.md` §AI1 自审清单（模糊表达排查）
